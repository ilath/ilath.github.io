编写可读代码的艺术

代码应当易于理解
=

1. 代码应当易于理解
2. 代码的写法应当使别人理解它所需的时间最小化
3. 尽管减少代码行数是一个好目标，但把理解代码所需的时间最小化是一个更好的目标
4. 要经常地想一想其他人是不是会觉得你的代码容易理解

第一部分 表面层次的改进
-

1. 选择好的名字
2. 写好的注释
3. 把代码整洁地写成更好的格式

把信息装到名字里
-
把信息塞入名字中。这句话的含义是，读者仅通过读到名字就可以获得大量信息。

  • 使用专业的单词
    1. 选择非常专业的词，并且避免使用“空洞”的词
    2. 找到更有表现力的词，例如：

      | 单词 | 更多选择 |
      | —- | ——- |
      | send | deliver, dispatch, announce, distribute, route |
      | find | search, extract, locate, recover |
      | start | launch, create, begin, open |
      | make | create, set up, bulid, generate, compose, add, new |

  • 避免空泛的名字

    1. 好的名字应当描述变量的目的或者它所承载的值
    2. 类似tmp这种名字只应用于短期存在且临时性为其主要存在因素的变量

  • 使用具体的名字来更细致地描述失误

    在给变量、函数或者其他元素命名时，要把它描述得更具体而不是更抽象

  • 给变量名带上重要的细节

    1. 为名字附带更多信息。如果关于一个变量有什么重要事情的读者必须知道，那么是值得把额外的“词”添加到名字中的
    2. 带单位的值。如果你的变量是一个度量的话，那么最好把名字带上它的单位

      | 函数参数 | 带单位的参数 |
      | —— | —— |
      | start(int delay) | delay -> delay_secs |
      | CreateCache(int size) | size -> size_mb |
      | ThrottleDowload(float limit) | limit -> max_kbps |
      | Rotate(float angle) | angle -> degrees_cw |

    3. 附带其他的重要属性。

  • 决定名字的长度

    1. 在小的作用域里可以使用短的名字
    2. “不好输入”不再是避免使用长名字的理由，要活用编辑器的“单词补全”
    3. 首字母缩略词和缩写。 原则：团队成员要能理解缩写的含义
    4. 丢掉没用的词。有时名字中的某些单词可以拿掉而不会损失任何信息，这种时候可以把这些单词去掉

  • 利用名字的格式来表达含义

    1. 对于下划线、连字符和大小写的使用方式也可以把更多信息装到名字中

候选样式表

将rel属性的值置为alternate stylesheet，就可以定义候选样式表，只有在用户选择这个样式表时才会用于文档表现。如果为一个rel为styleesheet的link指定了title，也就指定了该样式表要作为首选样式表。如果没有为样式表指定title，那么它将作为一个永久样式表，始终作用于文档的显示。

<link rel="stylesheet" type="text/css" href="sheet.css" title="Default"/>

<link rel="alternate stylesheet" type="text/css" href="sheet1.css"/>

属性选择器

简单属性选择

  • 简单属性选择器

    h1[class] { color: sliver; }

  • 元素属性选择器

    img[alt] { border: 3px solid red; }

    *[title] { font-weight: bold; }

    a[href][title] { font-weight: bold; }

根据具体属性值选择

  • 根据属性及其值选择元素

    这种格式要求必须与属性值完全匹配

    a[href="http://www.example.org"] { font-weight: bold; }

    a[href="http://www.example.org"][title="example"] { font-weight: bold; }

根据部分属性值选择

  • 根据部分属性值选择元素

    ~ 根据属性值中出现的一个用空格分隔的词来完成选择

    p[class~="warning"] { font-weight: bold; }

    这个选择器构造等价于点号类名记法。 p.warning 和 p[class~=”waring”] 是等价的。

  • 子串匹配属性选择器

    1. ^ [foo^="bar"]

      选择foo属性值以“bar”开头的所有元素

    2. $ [foo$="bar"]

      选择foo属性值以“bar”结尾的所有元素

    3. * [foo*="bar"]

      选择foo属性值中包含子串“bar”的所有元素

    其他用法示例：

    根据属性的值悬着元素

    a[href*="oreilly.com"] { font-weight: bold; }

特定属性选择类型

  • [att|="val"]

    这个规则会选择 att 属性等于val或者以 val- 开头的所有元素。

后代选择器

  • 选择子元素

    > 子结合符

    h1 > strong { color: red; }

    这个规则会把第一个h1下面出现的strong元素变成红色，但是第二个h1下出现的strong元素不会受影响。

  • 选择相邻兄弟元素

    + 相邻兄弟结合符

    h1 + p { margin-top: 0; }

    用一个结合符只能选择两个相邻兄弟中的第二个元素。另外，两个元素之间的文本内容不会影响相邻兄弟结合符起作用。
